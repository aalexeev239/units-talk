Хорошие юнит-тесты

  • Быстрые
    В вашем приложении будут тысячи тестов. Тысячи,{{nbsp}}Карл.
  • Изолированные
    Без зависимостей и сайд-эффектов
  • Повторяемые
    Тест должен быть предсказуем. Результат не{{nbsp}}изменится, пока вы что-то не поменяете в коде.
  • Уместные
    Тесты не должны писаться дольше основного кода
{{else}}

Good unit tests are

  • Fast
    There will be thousands of tests in your app. Thousands.
  • Isolated
    Units must be standalone, without any dependencies or side effects.
  • Repeatable
    Unit should be consistent with it result. The result shouldn't change unless you change something in code.
  • Timely
    Time for writing test should be comparable to writing code
